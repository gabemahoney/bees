# Sample Epic - E-commerce Platform

This epic demonstrates all the Epic-specific fields in the ticket system. It represents a user-testable feature that delivers end-user value.

## Description

Implement a complete e-commerce platform with the following capabilities:
- Product catalog with search and filtering
- Shopping cart with add/remove/update quantity
- Checkout flow with payment processing
- Order history and tracking

## Acceptance Criteria

- Users can browse products and search by name, category, or price
- Users can add products to cart and modify quantities
- Users can complete checkout with credit card payment
- Users can view their order history and track shipments

## Epic-Specific Fields

This sample demonstrates:
- **id**: A unique identifier (bees-ep1)
- **type**: Set to "epic" for top-level features
- **title**: Short, descriptive name
- **description**: Detailed explanation of the feature
- **labels**: Categorization tags (feature, backend, frontend)
- **up_dependencies**: Other tickets that block this epic (empty in this example)
- **down_dependencies**: Other tickets blocked by this epic (empty in this example)
- **children**: Tasks that implement this epic (will be populated when tasks reference this as parent)
- **status**: Current state (open, in progress, completed, etc.)
- **priority**: Importance level (0-4, where 0 is highest)
- **owner**: Team or individual responsible
- **created_at**: Timestamp when ticket was created
- **updated_at**: Timestamp of last modification
